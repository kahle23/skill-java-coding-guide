# 01 — 设计工作流

## 第 1 步：理解需求

明确以下信息：
- 要存储什么数据（实体是什么）
- 数据之间的关系（一对一 / 一对多 / 多对多）
- 核心业务字段有哪些
- 查询场景（高频查询字段决定索引设计）

## 第 2 步：查看历史风格

如果项目存在 `docs/table.sql`、`sql/` 目录或类似的历史 DDL 文件，**优先读取并分析**：
- 模块前缀惯例（如 `oa_`、`crm_`）
- 已有的标准字段风格
- 索引命名模式
- SQL 格式化风格

> 新表的风格必须与项目已有表保持一致。

## 第 3 步：设计表结构

按 `03-table-standards.md` 中的规范输出完整的 CREATE TABLE 语句。

## 第 4 步：补充说明

输出表结构后，同时提供：
1. **索引设计理由** — 为什么建这些索引（支撑哪些查询场景）
2. **字段填写说明** — 哪些必填、哪些有默认值、特殊字段的业务含义
3. **与现有表的关联关系** — 外键逻辑关联（不强制物理外键）
