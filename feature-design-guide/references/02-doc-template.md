# 02 — 设计文档模板

文档必须包含以下五个章节。

## 一、需求概述

- 功能背景与目标
- 适用范围说明（如：仅IT设备 或 全公司资产）
- 核心名词定义（如有业务专属术语）

## 二、功能模块梳理

用表格列出所有功能点：

| 模块 | 功能点 | 说明 | 优先级 |
|------|--------|------|--------|
|      |        |      | 核心/扩展 |

- 区分核心功能和扩展功能
- 按模块分组，逻辑清晰

## 三、表结构设计

> 表结构的具体规范（标准字段、命名、索引、禁止项等）由 `db-design-guide` 技能统一管理。本章节仅输出设计结果。

### 3.1 新建表

每张新表提供完整的 CREATE TABLE SQL，遵循 `db-design-guide` 规范。

### 3.2 复用现有表

| 表名 | 复用方式 | 说明 |
|------|----------|------|
|      | 直接读写 / 关联查询 |      |

### 3.3 数据字典预设（如需要）

提供 `sys_dict_group` / `sys_dict_item` 初始化 SQL。

## 四、接口梳理

按功能模块分组，每组一个表格：

| 序号 | 接口地址 | 请求方式 | 接口说明 | 关键入参 | 关键出参 |
|------|----------|----------|----------|----------|----------|

接口地址规范：
- 前缀使用 `/{模块标识}`，如 `/oa/asset/...`，不使用 `/api`
- RESTful 风格，资源名小写下划线
- 与表命名前缀保持一致

## 五、关键业务逻辑说明

对复杂业务逻辑进行详细说明：

- **编码/编号自动生成规则**：格式、递增策略、重置条件
- **状态流转规则**：建议用文本流程图表示状态变更路径
- **异步处理逻辑**：导入/导出/生成任务的执行流程和异常处理
- **自动触发机制**：如用户禁用自动解除关联、状态联动更新
- **数据权限控制**：按机构/角色/数据所属人的可见范围
- **字段填写说明**：不同场景下哪些字段必填、哪些可选
